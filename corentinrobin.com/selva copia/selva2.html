Introduction

The purpose of Selva is to update the UI of a single-page application in real time with the JavaScript values.

How it works: all selva-* attributes will be evaluated and set to the target attribute.

What the selva attribute can contain: a string, and anything in side {...} will be evaluated and replaced in the string.

For example, selva-style will set the style attribute, selva-src will change src.

The only exception is selva-value, which handles the content of the tag, and binds it to the variable if it's an input.

For example, doing selva-value="{firstName}", on an text input will bind the global variable firstName to this input.

If the variable is changed in JS, the input value will change, and if the user type a value through the UI, the variable will be updated.

Putting selva-value="Hello world!" to an h1, a p or any non-input tag will simply update the content.

The purpose of Selva is to keep things simple and keep a way of working as close as possible to HTML and JavaScript.

One consider that as it is an SPA, we don't use any namespace.

I think using multiple small and simple libraries is easier that using a big complicated one.

This is a small and simple library ;)
